# Start plan -- Thomasson SFL review

## Recommendation

Start with a narrow source-grounding pass, then draft the two load-bearing sections: the introduction and the grammatical-metaphor/category-function section.

The temptation is to begin by filling the manuscript from front to back. I think that is the wrong first move. The project already has a thesis and structure. What it lacks is a clean source packet: verified bibliographic entries, real page numbers, and a short passage map for the claims that will carry the review. Once that packet exists, the draft can move quickly without leaving hidden debt.

## Load-bearing assumptions

1. The linguistics-facing piece should stay anchored to *Rethinking Metaphysics* rather than becoming a general Thomasson article.
2. The contribution is Brett's Huddleston-line standpoint: enough SFL competence to understand the borrowing, and enough autonomist commitment to diagnose the placement error.
3. The central critical claim is stable: Thomasson imports the SFL function-in-syntax architecture in her vocabulary, but her conclusions mostly ride on easy ontology plus framework-neutral functional facts.
4. The companion philosophy piece carries the projectibility/kinds argument; this piece should mention it only where needed to keep boundaries clear.

## What would change the plan

1. If the page-number check weakens the line-ref claims in `notes/source-verification.md`, revise the thesis before drafting.
2. If *Norms and Necessity* turns out to be necessary for the linguistics-facing argument rather than just background, add a short bridge section or footnote early.
3. If the Halliday/Huddleston lineage cannot be independently sourced cleanly, keep the standpoint as Brett's methodological location and avoid historical overclaiming.
4. If the intended venue is *Functions of Language*, soften the autonomy-of-syntax critique's framing; if it is *Journal of Linguistics* or *Language Sciences*, keep it sharper.

## First work packet

1. Build `notes/passage-map.md`.

Capture each claim we expect to use, with local Markdown line references, PDF page numbers, and a note on how the passage functions in the argument. Start with:

- SFL as the engine of the positive program.
- Thomasson on not having discovered SFL when writing *Norms and Necessity*.
- Definition of grammatical metaphor.
- Eggins/Halliday nominalization passage.
- The abstracta/tracking-function argument.
- The chapter 9 concession about incompleteness and base forms.

2. Verify bibliography.

Add only verified entries to `references-local.bib`: Thomasson 2025, Thomasson 2020, Halliday 2009, Eggins 2004, plus Carnap only if the first draft actually cites him. Use authoritative metadata, not memory.

3. Draft Section 1.

Make the news concrete in the first two paragraphs: a leading metaphysician has built the positive program of a 2025 metaphysics book on Hallidayan SFL, and linguists should notice because this exports an internal linguistics architecture into metaphilosophy.

4. Draft Section 4.

Write the core diagnostic section before the expository middle. This is the argument's pressure test: grammatical metaphor makes category shift look like functional explanation; the useful point about nominalization survives, but the function-in-syntax architecture does not do the inferential work Thomasson needs.

5. Then draft Sections 2, 3, and 5.

Use the already-written Section 4 to discipline the exposition. The middle should explain only what the reader needs in order to understand the diagnostic claim.
